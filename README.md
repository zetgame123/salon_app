# Salon App

Клиент-серверное desktop-приложение для парикмахерской на JavaFX + MySQL.

## Требования

- **JDK 21**
- **Maven 3.8+**
- **MySQL 8** или **MariaDB 10.4+** (сервер должен быть запущен)

## 1. Развёртывание базы данных

### 1.1. Создание базы и пользователя

Подключитесь к MySQL под учётной записью с правами администратора (например, `root`) и выполните:

```sql
CREATE DATABASE salon
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_general_ci;

CREATE USER 'salon_app'@'localhost' IDENTIFIED BY '555666';

GRANT ALL PRIVILEGES ON salon.* TO 'salon_app'@'localhost';

FLUSH PRIVILEGES;
```

> Пароль `555666` совпадает с настройками по умолчанию в `database.properties`.  
> При смене пароля обновите и файл конфигурации приложения (см. раздел 2).

### 1.2. Импорт схемы

В корне проекта лежит скрипт `sql/salon.sql`. Он создаёт таблицы, внешние ключи, триггеры и представление `service_history`.

**Через командную строку:**

```bash
mysql -u salon_app -p salon < sql/salon.sql
```

**Через phpMyAdmin:**

1. Выберите базу `salon`
2. Вкладка «Импорт» → выберите файл `sql/salon.sql` → «Вперёд»

После импорта в таблице `roles` будут две записи: `ADMIN` и `USER`.

### 1.3. Тестовые данные (рекомендуется)

Скрипт `salon.sql` содержит только структуру. Для работы приложения нужны хотя бы филиалы, типы стрижек и цены. Выполните в базе `salon`:

```sql
INSERT INTO branches (name, address) VALUES
('Центральный', 'ул. Ленина, 1'),
('Северный', 'пр. Мира, 10');

INSERT INTO haircut_types (name, gender) VALUES
('Мужская стрижка', 'Мужской'),
('Женская стрижка', 'Женский'),
('Детская стрижка', 'Унисекс');

INSERT INTO prices (id_haircut_type, start_date, price) VALUES
(1, '2025-01-01', 500.00),
(2, '2025-01-01', 800.00),
(3, '2025-01-01', 400.00);
```

Либо добавьте данные через интерфейс приложения (разделы «Филиалы» и «Прайс», доступны ADMIN).

## 2. Подключение приложения к БД

Параметры подключения задаются в файле `database.properties`:

```properties
db.url=jdbc:mysql://localhost:3306/salon
db.user=salon_app
db.password=555666
```

| Параметр      | Описание                                      |
|---------------|-----------------------------------------------|
| `db.url`      | Адрес сервера, порт и имя базы                |
| `db.user`     | Имя пользователя MySQL                        |
| `db.password` | Пароль пользователя MySQL                     |

Если MySQL работает на другом хосте или порту, измените `db.url`, например:

```properties
db.url=jdbc:mysql://192.168.1.10:3306/salon
```

### Откуда читается файл

`DatabaseManager` ищет настройки в таком порядке:

1. **`database.properties` в рабочей папке** — откуда запущено приложение
2. **`database.properties` рядом с JAR** — в той же папке, что и `salon.jar`
3. **Встроенный файл** — `src/main/resources/database.properties` (используется при разработке в IDE, если внешнего файла нет)

При запуске в консоли будет указано, откуда загружены настройки:

```
Настройки БД загружены из: D:\salon\database.properties
Подключение к БД успешно.
```

### Разработка (IntelliJ IDEA)

Редактируйте файл:

```
src/main/resources/database.properties
```

### Запуск из JAR

После сборки JAR настройки **не нужно** пересобирать — достаточно внешнего файла:

```
папка_с_приложением/
├── salon.jar
└── database.properties    ← создайте из database.properties.example
```

1. Скопируйте `database.properties.example` → `database.properties`
2. Измените `db.url`, `db.user`, `db.password` под свою среду
3. Запустите JAR из этой же папки:

```bash
java -jar salon.jar
```

> Шаблон для копирования: `database.properties.example` в корне проекта.

## 3. Первый запуск и учётная запись ADMIN

В SQL-скрипте нет готовых пользователей. Порядок действий:

1. Запустите приложение (см. раздел 4)
2. На экране авторизации нажмите **«Регистрация»** и создайте пользователя  
   (пароль: мин. 8 символов, цифра, заглавная буква, спецсимвол, например `Admin123!`)
3. Назначьте роль ADMIN в базе данных:

```sql
UPDATE users SET id_role = 1 WHERE login = 'ваш_логин';
```

После этого войдите под этим логином — станут доступны разделы «Пользователи», «Филиалы», редактирование прайса и удаление записей.

## 4. Запуск приложения

### IntelliJ IDEA

1. Откройте проект как Maven-проект
2. Дождитесь загрузки зависимостей
3. Запустите класс `ru.ns.MainApplication`

### Maven (командная строка)

```bash
mvn clean compile
mvn javafx:run
```

Точка входа: `ru.ns.MainApplication`.

## 5. Сборка JAR

Для JavaFX-приложения используется **fat JAR** — один файл со всеми зависимостями (JavaFX, MySQL-драйвер, BCrypt).

### Сборка

```bash
mvn clean package
```

Готовый файл:

```
target/salon.jar
```

### Запуск JAR

1. Положите `database.properties` **в ту же папку**, что и `salon.jar` (см. раздел 2)
2. Убедитесь, что MySQL запущен и база развёрнута
3. Выполните:

```bash
java -jar salon.jar
```

**Windows (двойной клик):** положите `salon.jar` и `database.properties` в одну папку. Для запуска двойным кликом нужна установленная **JDK 21** (JRE без JavaFX не подойдёт).

### Разработка без JAR

```bash
mvn javafx:run
```

## 7. Возможные проблемы

**«Не найден файл database.properties»**  
При запуске из JAR положите `database.properties` рядом с JAR-файлом (см. раздел 2).  
При разработке — убедитесь, что файл есть в `src/main/resources/`.

**«Ошибка подключения к БД»**  
- MySQL/MariaDB запущен  
- База `salon` создана, скрипт `salon.sql` импортирован  
- Логин и пароль в `database.properties` совпадают с созданным пользователем `salon_app`

**«Заполните справочники: клиенты и филиалы»** (при добавлении услуги)  
Добавьте филиалы, типы стрижек и цены (см. раздел 1.3 или через UI ADMIN).

**Не удалось удалить клиента / филиал / цену**  
На запись ссылаются услуги в таблице `services` (ограничение внешнего ключа).
